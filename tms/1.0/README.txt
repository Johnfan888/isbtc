/********  本目录放置运输管理系统(tms)的V1.0代码  ********/

1、各子目录对应模块如下，请将各自负责模块的源码文件放在相应的目录里。可以根据需要在各模块目录下创建子模块目录，子模块目录命名采用拼音，自由命名，但需表述明确。

accouting	-->	营收结算模块
basedata	--> 	基础数据模块
checkin		--> 	检票管理模块
luggage		--> 	行包管理模块
querystatistic	--> 	查询统计模块
safecheck	--> 	安检管理模块
schedule	-->	调度管理模块
sell		--> 	售票管理模块
system		-->	系统管理模块

2、源码文件命名规范

所有文件都采用“项目名称_版本号_模块名称_功能名称（或子模块名称_功能名称）.扩展名”的命名方式，其中：项目名称_版本号_模块名称部分采用小写英文，固定命名；功能名称（或子模块名称_功能名称）部分采用拼音，自由命名（但需表述明确）；扩展名可以是php、conf（配置文件）、sql（数据库配置文件）等。示例如下：

(1)如1.0版的tms基础数据模块中的站点管理文件名为：tms_v1_basedata_zhandian.php （用功能名称）

(2)如1.0版的tms调度管理模块中的导乘子系统的LCD显示文件名为：tms_v1_schedule_daocheng_LCDxianshi.php (用子模块名称_功能名称）

(3)
如1.0版的tms售票管理模块中的配置文件名为：tms_v1_sell.conf

(4)
如1.0版的tms营收结算模块中的数据库配置文件名为：tms_v1_accouting.sql


3、编码规范参见docs\tms目录下《MS_CodingStyle.doc》文档
